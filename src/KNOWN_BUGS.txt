These are known bugs remaining after upgrading to GCC 8.1.1 and making quite
a lot of code changes to suppress (ok, fix) warnings generated by the compiler.

Two known issues...
Both issues are in the jobsched_cmd user interface, not in the job scheduler itself.

------------------------------------------------------------------------------------------
(1) MINOR - irritating
    The jobsched_cmd program does not upercase the calendar name on a job add command,
    the user must ensure the calendar name is uppercased on the command line

    (todo: correct jobsched_cmd to uppercase the calendar name in this job add instance)

== EXAMPLE ==

command:job add test-job3,owner mark,time 20181020 11:00,cmd /bin/ls,desc another test job,calendar sundays

*E* No entries found for next 5 years of entries

command:job add test-job3,owner mark,time 20181020 11:00,cmd /bin/ls,desc another test job,calendar SUNDAYS

JOB HAS BEEN ADDED

(2) MINOR - irritating and confusing
    When requesting calendar information the 'TYPE' of JOB or HOLIDAY must be provided
    and used to produce an error message when omitted.
    Now if omitted a partial response header is shown instead of the error message.
    The correct response is shown if the parameter is correctly provided...
    ...(so minor, if the correct command is used it works as expected)

    (todo: 
           (a) the server should check and discard the request with an error
               response. Do that first to accomadate future interfaces that
               may not set api fields correctly
           (b) sanity check input in jobsched_cmd and block/error there rather
               than bothering the server (after server checks have been tested.
    )
