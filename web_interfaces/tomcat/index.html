The job scheduler JSP interface is designed to provide a frontend to Marks Job Scheduler, © Mark Dickinson 2002, 2003, 2004 application. By providing the interface as a Tomcat JSP application I am able to provide full session authentication for authorised scheduler users, enabling the JSP code to implement the role segregation functions designed into the scheduler application itself; by performing authorisations against the scheduler databases and using Tomcats inherent session tracking ability.

It has also allowed the use of permanent re-useable socket connection pool entities which significantly cuts down on connection overheads.

Job Scheduler Highlights
  • Separation of user function by role
  • Job dependency on other jobs and files
  • Job and Holiday Calendar support
  • Runs jobs under multiple userids, full control of who can add jobs for a given userid
  • Apache operator interface, to monitor and control the active job queues of multiple servers from one screen
  • Tomcat interface to provide full control of all networked job schedulers
  • Full job audit trail history
  • Security reporting tools provided
